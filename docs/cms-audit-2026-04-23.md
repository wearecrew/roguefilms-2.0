# CMS field audit — 2026-04-23

Schema pulled fresh from the Webflow API. Goal: identify fields (or whole
collections) that can be marked `[DEPRECATED]` in helpText to clean up the
editor experience. **No fields will be deleted** per the non-destructive-CMS
principle in the README — the 700+ Showreels are live and we don't rename
or remove anything they depend on.

Summary: 9 collections total. Mostly clean. The main cruft is on the
**Director showreels** collection (a handful of fields made redundant by the
new `Homepage featured showreels` collection and the Phase 3 lightbox), plus
two legacy collections that look untouched since v1.

| vimeo video | `video-2` (VideoLink) | **Confirm first** | Was used for social share URLs in v1. Phase 3 lightbox copies the canonical `/director-showreels/[slug]` URL instead, so this should be redundant. But if any legacy page / schema.org block reads it, keep for now. |

| Everything else | — | **Keep** | `preview-image`, `brand`, `director`, `video-url`, `hero-video`, `is-anamorphic-ratio`, `video-width`, `video-height`, `work-status`, `work-type`, `project-stills`, `credits`, `film-poster-frame`, `show-film-credits` are all actively bound in Phase 2/3/4 pages. |
